1. Overview

SmartPM is a full-featured project management application purpose-built for ELV (Extra-Low Voltage) construction projects. It is developed by 3D Technology Services Inc. and designed to manage every stage of a project lifecycle — from importing AI-generated estimates out of SmartPlans to day-to-day tracking of materials installed, labor hours spent, billing, change orders, RFIs, infrastructure, and closeout.

SmartPM integrates directly with SmartPlans to provide a seamless estimate-to-execution workflow. Estimates created in SmartPlans can be exported as JSON and imported into SmartPM, automatically populating the project with SOV line items, WBS phases, infrastructure locations, equipment, cable runs, and RFIs.

Six metric cards are displayed at the top of the Dashboard:

  • "Import from SmartPlans" — Opens the SmartPlans import dialog (Admin / Ops Mgr only)

Deleting Projects

Admin users see a red "Delete Project" button on every project card. Clicking it shows a confirmation dialog, then permanently removes the project and all associated data (WBS, infrastructure, SOV, billing, change orders, RFIs, submittals, daily logs, punch items, contacts, and documents).

Deletion is permanent. There is no undo. Export any data you need before deleting a project.

5. SmartPlans Import

Importing from SmartPlans is the primary workflow for creating fully-populated projects. SmartPlans is the companion estimating tool that generates AI-powered ELV construction estimates.

How to Import

  1. On the Dashboard, click "Import from SmartPlans"
  2. Click "Choose JSON File" and select the SmartPlans export file
  3. Review the import preview — it shows the project name, contract value, SOV categories, infrastructure locations, WBS phases, and RFIs that will be created
  4. Click "Import Project"
  5. SmartPM creates the project with all data pre-populated from the estimate

What Gets Imported

SmartPlans DataSmartPM Destination
Project name, type, locationProject metadata
Pricing / contract valueProject settings and financial summary
SOV line items (BOM categories)Schedule of Values with material/labor cost breakdown
MDF/IDF/TR roomsInfrastructure locations
Equipment per roomLocation equipment items (locked budgets)
Cable runsCable run records with budgeted footage
WBS phases and tasksWork Breakdown Structure hierarchy
Selected RFIsRFIs module (auto-populated)
Budgeted amounts imported from SmartPlans are locked. Only Admins and Ops Managers can modify budgets. PMs can only update actual quantities and hours.

Re-importing (Updating an Existing Project)

If the SmartPlans estimate is updated after the initial import, you can re-import to refresh the project data:

  1. Open the project and go to the Overview tab
  2. Click "Re-import from SmartPlans" (only visible for projects that were originally imported)
  3. Select the updated JSON export file
  4. The re-import updates budgeted data while preserving any field-level changes (actuals, progress, notes) that were entered in SmartPM
The re-import button only appears on projects that have a smartplans_import_id — meaning they were originally created via SmartPlans import.

7. Schedule of Values (SOV)

The SOV module manages AIA G703-format line items for the project. Each line item has a scheduled value with material and labor cost breakdown from the SmartPlans BOM, along with a completion percentage.

  • Line items are auto-created when importing from SmartPlans
  • Admin/Ops Mgr users can add, edit, or delete SOV line items manually
  • Each item tracks: item number, description, scheduled value, material cost, labor cost, and percent complete
Use the AI SOV Validation feature to cross-reference reported completion percentages against daily log evidence.

8. Progress Billing

The Progress Billing module tracks payment applications per billing period (typically monthly).

  • Create billing periods and record work completed and materials stored for each SOV line item
  • The system calculates retainage, previous payments, and current payment due
  • Each billing period links to the SOV for consistent tracking

9. Work Breakdown Structure (WBS)

The WBS module displays the entire project broken down into a Phase / Location / Task hierarchy, auto-generated from the SmartPlans bid.

WBS Dashboard Metrics

  • Project Health — Traffic light indicator (green = on track, yellow = caution, red = over budget)
  • Total WBS Tasks — Count of all tasks in the hierarchy
  • Overall Progress — Weighted percentage complete
  • Material Budget — Actual vs. budgeted material cost with traffic light
  • Labor Budget — Actual vs. budgeted labor hours with traffic light
  • Tasks Complete — Count of completed tasks vs. total

Phase Filtering

Filter buttons let you view tasks by construction phase: All Phases, Rough-In, Trim-Out, Termination & Test, or Closeout.

Editing a WBS Task

Click the edit button on any task to open the update modal:

11. Change Orders

Track scope changes with financial and schedule impact throughout the project.

  • Types: Addition, Deduction, No-Cost Change
  • Status flow: Pending → Approved / Rejected
  • Each change order records: description, cost amount, schedule impact (days), and justification
  • Approved change orders adjust the current contract value on the project overview
  • AI Feature: Click "AI Impact Analysis" for a cumulative financial and schedule impact assessment across all change orders

12. RFIs (Requests for Information)

Track Requests for Information from submission through resolution.

  • RFIs are auto-populated from SmartPlans import when the estimate includes flagged questions
  • Create new RFIs manually at any time during the project
  • Status flow: Open → Answered → Closed
  • Each RFI tracks: subject, question, response, assigned to, date submitted, and date answered
  • AI Feature: "AI Smart RFIs" scans your project data (SOV, infrastructure, daily logs) and drafts professional RFI questions for items that need clarification

13. Submittals

The Submittal module provides a log for tracking submittals sent to the architect or engineer for approval.

  • Create submittal entries with number, description, spec section, and status
  • Track submittal lifecycle: Pending → Submitted → Approved / Rejected / Revise & Resubmit
  • Record dates for submission, review, and response

14. Daily Log

Record daily field reports for each working day on the project.

  • Weather conditions and temperature
  • Crew size and hours worked
  • Work performed (with specific detail — room numbers, equipment, cable counts)
  • Delays, safety incidents, and visitor log
  • Materials received and installed
Pro Tip: Be specific in daily logs — mention room numbers, equipment models, and cable counts. The AI Assistant uses your daily log entries to generate progress reports and productivity analysis.

16. Punch List

Track deficiency items identified during walkthroughs, from discovery through closeout verification.

  • Assign items by location, discipline, and priority level
  • Status flow: Open → In Progress → Completed → Verified
  • Record responsible party, due date, and resolution notes
  • AI Feature: "AI Priority Plan" ranks open punch items by criticality (safety → code → functional → cosmetic) and generates a day-by-day action plan for efficient closeout

17. Contacts

The Contacts module provides a directory of project stakeholders — owners, GC representatives, architects, engineers, subcontractors, and inspectors.

  • Add contacts with name, company, role, phone, and email
  • Contacts can be scoped to a specific project or shared across all projects
  • Edit or delete contacts as the project team changes

18. Document Manager

Upload, organize, and download project documents such as specifications, shop drawings, photos, and submittal packages.

  • Upload documents with category tags for organization
  • Drag-and-drop support for quick file uploads
  • Category filtering to find documents quickly
  • File type icons displayed based on the document format
  • Download any document to your local machine
  • Delete documents that are no longer needed
Documents are stored in the cloud database. The API supports both uploading (POST /api/projects/:id/documents) and downloading (GET /api/projects/:id/documents/:docId) with optional metadata-only retrieval.

20. User Management (Admin Only)

User account management is available from the header navigation bar for Admin and Ops Manager roles.

  • Admin can change passwords for all users: Admin, Ops Mgr, PM, and Viewer (3D)
  • Ops Manager can change the PM password only
  • Passwords must be at least 8 characters and are confirmed with a re-entry field

Password Security

All passwords are hashed using PBKDF2 with a unique per-user salt before storage. Plaintext passwords are never stored in the database.

22. Health Monitoring

SmartPM provides a health check endpoint for uptime monitoring and diagnostics.

Health Endpoint

GET /api/health

Returns the application status including:

  • Database connectivity status
  • Table count in the database
  • Total number of users and projects

Database Migrations (Admin)

  • GET /api/migrations — Check the status of pending database migrations
  • POST /api/migrations — Run any pending migrations to update the database schema
Migrations are idempotent and safe to run multiple times. They are typically needed after a SmartPM version upgrade.

23. Troubleshooting

IssueSolution
Can't see my projectsProjects are scoped to the user who created them. Only Admin users can see all projects. Verify you are logged into the correct account.
Import failsEnsure the JSON file is a valid SmartPlans export. The file must contain a _meta.format field identifying it as a SmartPlans export. Check the browser console for detailed error messages.
Re-import button missingThe "Re-import from SmartPlans" button only appears for projects that were originally imported from SmartPlans (those with a smartplans_import_id). Manually created projects cannot be re-imported.
Login failsCheck with your Admin for correct credentials. Passwords are case-sensitive. If your session expired, you will be automatically redirected to the login screen.
API rate limitedWait 60 seconds and try again. Heavy endpoints (AI features, SmartPlans import) have stricter rate limits than standard operations.
Budget fields are lockedBudgeted amounts from SmartPlans are locked by design. Only Admin and Ops Manager roles can modify budget fields. PMs can edit actual quantities and hours only.
AI analysis failsEnsure you have project data (SOV items, daily logs, change orders, etc.) for the AI to analyze. Empty projects will return errors.
Daily report won't emailThe email button uses mailto: links. You need an email client configured on your device. Alternatively, use "Copy to Clipboard" and paste into any email.
Data not savingCheck your internet connection. SmartPM requires connectivity to save to the cloud database. An offline banner appears when connectivity is lost.
Can't delete a projectOnly Admin role can delete projects. Contact your Admin if a project needs to be removed.
